Lambda表达式

Java 8的一个大亮点是引入Lambda表达式，使用它设计的代码会更加简洁。当开发者在编写Lambda表达式时，也会随之被编译成一个函数式接口。使用Lambda表达式不仅让代码变的简单、而且可读、最重要的是代码量也随之减少很多。

演变过程

垃圾代码 –> 策略模式 –> 匿名内部类 –> Lambda表达式

举例

  • 匿名内部类

    Comparator<Integer> comparator = new Comparator<Integer>() {
    @Override
    public int compare(Integer o1, Integer o2) {
        return Integer.compare(01,02);
    }
};

  • Lambda表达式

    Comparator<Integer> comparator = (a,b) -> Integer.compare(a,b);

格式

  • 无参数无返回值

    public class Test02 {
	int num = 10; //jdk 1.7以前 必须final修饰
    
    @Test
    public void test01(){
        //匿名内部类
        new Runnable() {
            @Override
            public void run() {
                //在局部类中引用同级局部变量
                //只读
                System.out.println("Hello World" + num);
            }
        };
    }

    @Test
    public void test02(){
        //语法糖
     	Runnable runnable = () -> {
         	System.out.println("Hello Lambda");
     	};
    }
}

  • 有一个参数无返回值

    @Test
public void test03(){
    Consumer<String> consumer = (a) -> System.out.println(a);
    consumer.accept("我觉得还行！");
}

  • 有两个以上的参数，有返回值

    @Test
public void test04(){
    Comparator<Integer> comparator = (a, b) -> {
        System.out.println("比较接口");
        return Integer.compare(a, b);
    };
}

  • 有两个以上的参数，有返回值，并且 Lambda 体中只有1条语句 （大括号 与 return 都可以省略不写）

    @Test
public void test04(){
    Comparator<Integer> comparator = (a, b) -> Integer.compare(a, b);
}

函数式接口

定义一个接口，只能存放一个方法，并且要添加注解 @FunctionalInterface

定义一个接口，可以不使用泛型

@FunctionalInterface
public interface MyFun <T>{
    T count(T t, T r);
}

测试

MyFun<Integer> myFun = (x,y)-> x+y;
MyFun<Integer> myFun2 = (x,y)-> x-y;
MyFun<String> myFun3 = (x,y)-> x+y;
MyFun<Double> myFun4 = (x,y)-> x*y;

Integer count = myFun.count(1, 2);
System.out.println(count);
Integer count2 = myFun2.count(1, 2);
System.out.println(count2);
String count3 = myFun3.count("1", "2");
System.out.println(count3);
Double count4 = myFun4.count(1.1,2.1);
System.out.println(count4);

/*
结果是：
    3
    -1
    12
    2.3100000000000005
*/

可以发现，函数式接口让我们在实现业务的时候有了很高的自由度，代码书写也更加的紧凑，简便，减少了代码量

函数式编程

Java内置四大核心函数式接口：

函数式接口 参数类型 返回类型 用途
Consumer
消费型接口		 T void 对类型为T的对象应用操作：void accept(T t)
Supplier
提供型接口		 无 T 返回类型为T的对象：T get()
Function<T, R>
函数型接口		 T R 确定类型为T的对象是否满足某约束，并返回boolean值：boolean test(T t)
Predicate
断言型接口		 T boolean 确定类型为T的对象是否满足某约束，并返回boolean值：boolean test(T t)

消费型接口

@org.junit.Test
public void test06(){
    Consumer<String> consumer = (x)-> System.out.println(x);
    consumer.accept("123");
}

提供型接口

@org.junit.Test
public  void test07(){
    Supplier<List<Integer>> supplier = ()->{
        List<Integer> list = new ArrayList<>();
        list.add((int)(Math.random()*100));
        list.add((int)(Math.random()*100));
        list.add((int)(Math.random()*100));
        list.add((int)(Math.random()*100));
        list.add((int)(Math.random()*100));
        return list;
    };

    List<Integer> list = supplier.get();
    list.forEach(System.out::println);
}

函数型接口

@org.junit.Test
public void test08(){
    Function<Integer,Integer> function = (x)-> x+1;
    int apply = function.apply(1);
    System.out.println(apply);
}

断言型接口

@org.junit.Test
public void test09(){
    Predicate<String> predicate = x-> "predicate".equals(x);
    System.out.println(predicate.test("1"));
}

其他的接口

方法引用

如果Lambda体中的内容有方法已经实现了，我们可以使用方法引用(是Lambda的另一种表达形式)

语法格式

  • 对象::实例方法名

    @org.junit.Test
public void test1(){
    Consumer<String> consumer = (x)->System.out.println(x);
    /*
    两者都没有返回值，参数列表也一样
    对象::实例方法名
     */
    Cons<String> consumer2 = System.out::println;
}

@org.junit.Test
public void test2(){
    Users users = new Users();
    /*
    getPassword()的返回类型是String
    函数式接口中抽象方法的返回值也是String
    二者都没有参数
    可以简化lambda表达式
    */
    Supplier<String> supplier = users::getPassword;
    System.out.println(supplier.get());
}

  • 类::静态方法名

    @org.junit.Test
public void test3(){
    // 类:: 静态方法
    Comparator<Integer> comparator = (x,y)->Integer.compare(x,y);
    Comparator<Integer> comparator1 = Integer::compare;
    System.out.println(comparator1.compare(2, 1));
}

  • 类::实例方法名

      @org.junit.Test
  public void test4(){
      // 类:: 实例方法名
      BiPredicate<String,String> predicate = (x,y)->x.equals(y);
// 这一个跟上边的 类::静态方法名差不多
      BiPredicate<String,String> predicate2 = String::equals;
  }

始终要注意的是：

  • Lambda体黄总调用方法的参数列表和返回值类型，与函数式接口中抽象方法和返回值类型保持一致

  • 如果Lambda参数列表中的第一参数是实例方法的调用着，而第二个参数是实例方法的参数，可以用ClassName::method表示

构造器引用

格式

  • ClassName::new
/*
	简单案例，引用无参构造器
*/
@org.junit.Test
public void test5(){
    // className::new
    Supplier<Users> supplier = Users::new;
    System.out.println(supplier.get());
}

注意

  • 需要调用的构造器的参数列表要与函数式接口中抽象的参数列表保持一致

数组的引用

@org.junit.Test
public void test6(){
    Function<Integer,String[]> fun = (x)->new String[x];

    /*可以变为*/
    Function<Integer,String[]> fun2 = String[]::new;
}

强大的Stream API

流是数据渠道，用于操作数据源，数组，集合等生产的元素 序列

  • Stream 自己不会存储元素
  • Stream 不回改变源对象，他们会返回一个持有结果的新的Stream
  • Stream 操作是延迟执行的。这意味着他们会等到需要结果的时候才执行

步骤

  • 创建Stream
  • 中间操作
  • 终止操作

创建Stream

创建Stream的方法有多种

  • Connection接口下提供的方法stream()或者parallelStream()（并行流）

    // 通过Connection提供的steam()或者parallelStream()
List<String> list = new ArrayList<>();
Stream<String> stream = list.stream();

  • 通过Arrays中的静态方法获取

    Users[] users = new Users[10];
Stream<Users> stream1 = Arrays.stream(users);

  • Stream类的静态方法of()获取

    Stream<Integer> integerStream = Stream.of(1, 2, 1);

  • 创建无限流

    Stream<Integer> iterate = Stream.iterate(0, (x) -> x + 2);

    中间操作

Stream的中间操作和终端操作如下图所示：

切片和筛选

  • filter 接收Lambda,从流中排除某些元素
  • limit 截断流，使元素不超过给定数量
  • skip(n) 跳过元素，返回一个扔掉了前n个元素的流，如果不足n个则返回一个空流
  • distinct 筛选，通过流所产生元素的hashCode()和equals()去除重复元素
 @Test
 public void test1(){

     Stream<Users> stream = list.stream();
     /*
     * 中间操作:不会执行任何操作
     * 终止操作:一次性执行全部内容，即"惰性求值"
     
filter(Predicate<? super T> predicate)
满足predicate的条件的会被保留
     * */
     stream.filter((e)->e.getPassword().equals("123"))
     .forEach(System.out::println);
 }

 @Test
 public void test2(){
     Stream<Users> stream = list.stream();
     /*
     limit(n) 表示取前n个
     */
     stream.limit(2)
             .forEach(System.out::println);
 }

 @Test
 public void test3(){
     /*
     skip(n) 表示跳过前n个，配合limit可以有奇效
     */
     Stream<Users> stream = list.stream();
     stream.skip(2)
             .forEach(System.out::println);
 }

 @Test
 public void test4(){
     Stream<Users> stream = list.stream();
     /*
     去重操作需要比较二者的hashCode()和equals()，需要重写这两个方法
     */
     stream.distinct()
             .forEach(System.out::println);
 }

映射

  • map：接收 Lambda ，将元素转换为其他形式或提取信息；接受一个函数作为参数，该函数会被应用到每个元素上，并将其映射成一个新的元素
  • flatMap：接收一个函数作为参数，将流中每一个值都换成另一个流，然后把所有流重新连接成一个流(类似于collection中的addAll(Collection coll)方法)
@Test
public void test5(){
    List<String> list = Arrays.asList("111", "bbb", "ccc", "awdaw", "123");

    /*
    * map 接受一个函数作为参数，该函数会被应用到每个元素上，并将其映射成一个新的元素
    * */
    list.stream()
            .map(str->str.toUpperCase())
            .forEach(System.out::println);

    System.out.println("=============================");
    this.list.stream()
            .map(users -> users.getUsername().split("qweq"))
            .distinct()
            .forEach(System.out::println);

    System.out.println("=============================");
    /*
    *  flatMap 把所有流重新连接成一个流，即相同类型的流进行合并
    * */
    list.stream()
            .flatMap(TestStream::getStream)
            .forEach(System.out::println);
}

排序

  • sorted() 自然排序
  • sorted(Comparator c) 定制排序
@Test
public void test11(){
    List<String> list = Arrays.asList("111", "bbb", "ccc", "awdaw", "123");
    /*自然排序*/
    Stream<String> stream = list.stream();
    stream.sorted()
            .forEach(System.out::println);

    /*定制排序*/
    this.list.stream()
            .sorted((x,y)-> x.getUsername().compareTo(y.getUsername()))
            .forEach(System.out::println);
}

终止操作

查找与匹配

  • allMatch：检查是否匹配所有元素
  • anyMatch：检查是否至少匹配一个元素
  • noneMatch：检查是否没有匹配所有元素
  • findFirst：返回第一个元素
  • findAny：返回当前流中的任意元素
  • count：返回流中元素的总个数
  • max：返回流中最大值
  • min：返回流中最小值
public enum Status {
    FREE, BUSY, VOCATION;
}

@Test
public void test01(){
    List<Status> list = Arrays.asList(Status.FREE, Status.BUSY, Status.VOCATION);

    boolean flag1 = list.stream()
        .allMatch((s) -> s.equals(Status.BUSY));
    System.out.println(flag1);

    boolean flag2 = list.stream()
        .anyMatch((s) -> s.equals(Status.BUSY));
    System.out.println(flag2);

    boolean flag3 = list.stream()
        .noneMatch((s) -> s.equals(Status.BUSY));
    System.out.println(flag3);

    // 避免空指针异常
    Optional<Status> op1 = list.stream()
        .findFirst();
    // 如果Optional为空 找一个替代的对象
    Status s1 = op1.orElse(Status.BUSY);
    System.out.println(s1);

    // 随便获取一个
    Optional<Status> op2 = list.stream()
        .findAny();
    System.out.println(op2);

    long count = list.stream()
        .count();
    System.out.println(count);
}

归约与收集

  • 归约：reduce(T identity, BinaryOperator) / reduce(BinaryOperator) 可以将流中的数据反复结合起来，得到一个值
  • 收集：collect 将流转换成其他形式；接收一个 Collector 接口的实现，用于给流中元素做汇总的方法
