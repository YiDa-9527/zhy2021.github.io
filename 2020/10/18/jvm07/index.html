String Table

String的基本特性

String是final的，不可被继承，不可变

String的创建方式

  • String s1 = “zhanghaoyunb”; // 字面量定义方式
  • String s2 = new String(“hello”); new实例化的方式

String的底层存储是char类型的数组，这个是1.9之前的，在1.9的时候用byte[]来存储数据,节约了空间

String 代表不可变的字符序列：

@Test
public void test1(){
    String s1 = "123"; //字面量定义的方式，"123"存储在字符串常量池中
    String s2 = "123";
    s1 = "hello";

    System.out.println(s1==s2); // false
    System.out.println(s1);
    System.out.println(s2);
}
  • 字符串常量池中是不会存储相同内容的字符串的
  • String的String pool 是一个固定大小的Hash Table，jdk6默认值大小长度是1009。如果放进String Pool的String过多，就会造成hash冲突，导致很长的链表。从而影响了String,intern时候的性能。jdk7默认大小60013，jdk8开始，1009是可以设置的最小值

测试：

把10w条数据插入字符串常量池中，1009的长度，插入时间为147ms。100009的长度，插入时间为43ms

String的内存分配

String类型的常量池主要的使用方法有两种：

  • 直接使用“”声明出来的String对象会直接存储在常量池中：String info = “我被直接放在常量池中”
  • 如果不是“”声明的对象，可以使用String提供的intern（）方法。（重点）

历史

  • java 6以前字符串常量池放在永久代

  • java 7中字符串常量池放在堆中
    • 字符串都保存在堆中，这样调优时仅仅调整堆大小就可以了
    • 字符串常量池概念原本使用的比较多，但是这个改动使得我们有足够的理由让我们重新考虑在java 7中使用String,intern()
  • java 8 永久代改名为元空间，字符串常量还是在堆中

String Table为什么要调整

  • 防止永久代内存溢出
  • 永久代垃圾回收频率低

String的基本操作

用“”创建的字符串只会在字符串常量池中创建一次

@Test
public void test4(){
    System.out.println("1");
    System.out.println("2");
    System.out.println("1"); // 因为之前创建过了，所以这里使用的是之前创建的
}

字符串拼接操作

  1. 常量与常量的拼接结果在常量池，原理是编译器优化

    String s1 = "a"+"b"+"c";// 等同于 abc
String s2 = "abc"; // abc 一定是放在字符串常量池中，将地址赋值给s2
/*
常量与常量进行拼接，结果放入常量池，原理是编译期优化
字节码反编译后：
String s1 = "abc";
String s2 = "abc";

 */
System.out.println(s1 == s2);// true
System.out.println(s1.equals(s2));// true

  2. 常量池中不会存在相同内容的常量

    String s1 = "javaEE";
String s2 = "javaEE";
sout(s1==s2) // true
    
    
 String s1 = "javaEE";
String s2 = "hadoop";
String s3 = "javaEEhadoop";
String s4 = "javaEE"+"hadoop";  // 编译器优化
String s5 = s1 + "hadoop";  // 拼接符号前后出现了变量，则需要在堆空间中new一个，具体内容为拼接结果
String s6 = "javaEE" + s2;
String s7 = s1 + s2;

System.out.println(s3==s4); // true 常量相加结果放入常量池
System.out.println(s3==s5); // false
System.out.println(s3==s6); // false
System.out.println(s3==s7); // false
System.out.println(s5==s6); // false
System.out.println(s5==s7); // false
System.out.println(s6==s7); // false
// intern() 如果常量池中没有这个字符串，就会添加一个进去，并返回此对象（s6）的地址
// 如果有，就返回常量池中的地址
String s8 = s6.intern();
System.out.println(s3==s8); // true

  3. 只要其中有一个是变量，结果就在堆中。变量拼接的原理是StringBuilder 

    String s1 = "a";
String s2 = "b";
String s3 = "ab";
/*
如下的s1 + s2操作执行细节：

① StringBuilder s = new StringBuilder();
② s.append("a")
③ s.append("b")
④ s.toString() ---->底层实际上是new了一个String，返回回去

 */

+ 和 append的效率问题

append的执行效率要远远高于 字符串拼接的效率

字符串拼接底层，要先new一个StringBuilder，完了调用append方法，最后再调用toString方法返回一个字符串对象。

提高

我们一般在项目中，如果遇到多次字符串拼接，我们会创建StringBuilder，用append方法来代替字符串拼接操作。

当然还有一种更好的优化：因为StringBuilder构造器也是用默认长度16的数组构造的，所以

  1. 如果拼接的结果调用intern（）方法，则主动将常量池中还没有的字符串对象放入池中，并返回此对象的地址。

    我们可以来做一个验证，在上一个例子中，由于s6的 “javaEEhadoop” 在之前就存在了常量池中，所以只是把常量池中的对应字符串地址返回了

    s4是在堆空间中new 的String对象，存放的 "ab"，他们不是一个东西，所以是false

intern()的使用

String s = new String("1"); 
s.intern();
String s2 = "1";
System.out.println(s == s2);   ///?????????

String s3 = new String("1") + new String("1");
s3.intern();
String s4 = "11";

System.out.println(s3 == s4);   ////??????????????

概述

intern方法会判断当前字符串，在常量池中是否存在。如果存在就返回字符串常量池中的地址，如果不存在就会将当前字符串放入常量池中，并返回这个地址。这个要因为版本的不同，会有不同的情况，具体看下面的题

  • String s = “hello world” // 定义字面量方式
  • String s = new String(“xxxx”).intern();
  • String s = new StringBuilder(“xxxxx”).toString().intern();

然后我们来说一下本模块的引子，面试题：

String s = new String("1"); 
s.intern();     // 在这之前字符串常量池中已经存在1了
String s2 = "1";
System.out.println(s == s2);   ///  jdk6  false     jdk7/8 false

String s3 = new String("1") + new String("1");
s3.intern(); 
/*
jdk1.6  在字符串常量池中创建一个11对象
jdk7/8  在字符串常量池中开辟一个空间，记录堆中new的11的地址，所以s3 == s4，因为他们都指向的同一个对象
*/
String s4 = "11";

System.out.println(s3 == s4);   ////      jdk6 false    jdk7/8 true

解释：

​ new String(“1”)，根据我们之前的知识，可以知道他创建了两个对象，一个放在字符串常量池中，一个在堆中。当再次调用intern()方法的时候，由于字符串常量池中已经有了 “1”这个值，返回常量池中的地址，却没有接收对象，所以相当于没有做任何操作。s2 = “1” 就直接去字符串常量池中找，因为之前存在，返回原来的地址。

​ new String(“1”) + new String(“1”) ， 根据我们之前的知识，可以知道，字符串常量池中会存放 “1”，堆空间中会存放最后的结果 “11”。当调用 s3.intern()方法的时候。有以下情况：

  • jdk6的时候，jvm会在字符串常量池中new 一个新的对象，来保存11 ，同时堆空间中也存在一个11,s3指向的是堆空间的那个new 出来的，s4创建的时候，字符串常量池中已经存在了，所以把字符串常量池中的给了s4，所以他们不是同一个对象，故false
  • jdk7/8的时候，jvm会在字符串常量池中开辟空间，这时候不是保存”11”了，而是保存堆空间中new的”11”对象的地址。所以，s4创建的时候，字符串常量池中确实存在”11”这个东西，但是他实际上存放的是堆空间中的那个”11”的地址。所以这时候s4 和 s3 实际上是指向同一块内存地址，即堆空间中的”11”对象。

面试题拓展：

 String s3 = new String("1") + new String("1");
String s4 = "11";
s3.intern();      // 此时常量池中已经有了11，所以相当于啥也没干
System.out.println(s3 == s4);
